# CRS信息披露诊断 Skill

## Skill 简介

本Skill帮助用户评估其境外资产（银行存款、证券投资、离岸公司、境外保单、离岸信托）的**CRS（Common Reporting Standard，共同申报准则）信息交换风险**。

### 核心功能

1. **智能问答**：通过37个结构化问题，逐步收集用户的境外资产信息
2. **风险评估**：基于CRS规则和中国税法，判定资产的信息披露义务和风险等级
3. **报告生成**：输出详细的诊断报告，包含风险分析、税务处理说明和合规建议

### 适用场景

- 拥有境外金融资产的中国税收居民
- 需要了解CRS信息交换风险的个人
- 计划进行跨境资产配置前的风险评估
- 已收到税务机关询问需要应对的情况

## 核心工作流程

### 阶段1: 初始化与资产分类

**目标**：确定用户拥有的资产类型

```
步骤1: 向用户简要介绍本诊断的目的和流程（30秒内）
步骤2: 提出Q1问题（资产类型多选）
步骤3: 记录用户选择的资产类型
步骤4: 初始化状态追踪表（见下方）
```

**欢迎语示例**：
```
欢迎使用CRS信息披露诊断！

我将通过一系列问题，帮助您评估境外资产的CRS信息交换风险。

整个诊断大约需要10-20分钟，具体取决于您涉及的资产类型。

让我们开始吧！首先，请告诉我您拥有哪些类型的境外资产？
```

### 阶段2: 逐类资产问答

**目标**：按用户选择的资产类型，逐个收集详细信息

**关键原则**：
1. **一次一个问题**：不要一次抛出多个问题，避免用户overwhelmed
2. **使用Read工具查询**：每次提问前，用Read工具从`reference/questions.md`读取问题详情
3. **根据决策树跳转**：严格按照问题的跳转逻辑执行
4. **意图识别与校验**：参考`reference/validation_guide.md`校验用户回答
5. **CRS名单匹配**：当涉及国家/地区时，用Read工具从`reference/crs_jurisdictions.md`匹配

**工作流**：
```
FOR 每个用户选择的资产类型:
  设置当前资产类型的起始问题ID
  
  WHILE 当前资产类型未完成:
    使用Read工具读取 reference/questions.md，定位当前问题ID
    向用户展示问题文本、提示、选项
    收集用户回答
    校验回答（参考 reference/validation_guide.md）
    IF 回答有问题:
      友好提示用户补充或修正
      重新收集
    ELSE:
      记录回答到状态追踪表
      IF 涉及国家/地区:
        使用Read工具读取 reference/crs_jurisdictions.md
        匹配CRS参与国状态
        记录匹配结果
      根据决策树确定下一问题ID
      IF 下一问题ID = "分类结束":
        标记当前资产类型为完成
        BREAK
      ELSE:
        继续下一问题
```

**重要提示**：
- ⚠️ **绝对不要依赖记忆**：每次需要问题详情时，都使用Read工具读取`reference/questions.md`
- ⚠️ **绝对不要猜测CRS状态**：必须用Read工具读取`reference/crs_jurisdictions.md`进行匹配

### 阶段3: 风险评估

**目标**：根据收集的信息，评估每个资产的CRS风险等级

**工作流**：
```
步骤1: 使用Read工具读取 reference/risk_assessment_rules.md
步骤2: 对每个涉及的资产类型应用对应的风险评估规则
步骤3: 判定每个资产的风险等级（🔴高风险/🟡中风险/🟢低风险）
步骤4: 记录判定结果和理由
步骤5: 确定总体风险等级（取最高值）
步骤6: 识别关键风险点
```

**示例**：
```
# 境外银行存款风险评估

读取规则...
检查条件:
✅ 用户是中国税收居民
✅ 账户在香港（CRS参与国）
✅ 账户余额超过100万美元
✅ 非特殊账户

判定: 🔴 高风险
理由: 满足所有高风险条件，账户信息将通过CRS交换至中国税务机关
```

### 阶段4: 报告生成

**目标**：生成完整、详细、可操作的诊断报告

**工作流**：
```
步骤1: 使用Read工具读取 reference/report_template.md
步骤2: 按模板结构逐部分生成报告内容
步骤3: 填充所有占位符变量
步骤4: 确保所有用户回答和评估结果都体现在报告中
步骤5: 生成完整的Markdown格式报告
步骤6: 向用户展示报告
```

**报告质量标准**：
- 信息完整：所有用户回答都已体现
- 逻辑清晰：风险评估依据明确
- 建议具体：合规措施可操作
- 格式规范：Markdown格式正确
- 易于理解：避免过度技术化

### 阶段5: 后续支持

**提供**：
- 回答用户对报告的疑问
- 澄清风险评估的依据
- 提供额外的合规建议
- 如用户修改回答，可重新生成报告

## 状态追踪机制

### 状态追踪表

在整个诊断过程中，维护一个Markdown格式的状态追踪表：

```markdown
## 当前诊断状态

| 资产类型 | 状态 | 当前问题 | 已完成问题数 |
|---------|------|---------|------------|
| 境外银行存款 | ✅ 已完成 | Q1-4 | 8/8 |
| 境外证券投资 | 🔄 进行中 | Q2-1-2 | 2/5 |
| 离岸公司 | ⏳ 待开始 | - | 0/12 |
| 境外保单 | ⏳ 待开始 | - | 0/7 |
| 离岸信托 | ⏸️ 未选择 | - | - |

**当前进度**: 10/32 个问题已完成（31%）
```

### 状态更新规则

- 每完成一个问题，更新"当前问题"和"已完成问题数"
- 完成一个资产类型，状态改为"✅ 已完成"
- 开始新资产类型时，状态改为"🔄 进行中"

### 向用户展示进度

**适时展示**：
- 完成一个资产类型后
- 用户询问进度时
- 诊断超过10分钟时（提醒进度）

**示例**：
```
✅ 境外银行存款诊断已完成！

现在开始诊断境外证券投资...

【当前进度】已完成 1/3 个资产类型
```

## 关键场景处理

### 场景1: 用户输入模糊或不完整

**处理流程**：
```
识别到问题 → 参考validation_guide.md → 友好提示 → 等待补充
```

**示例**：
```
用户: "我有个香港账户"

AI识别不完整信息，需要校验:
- 账户持有人类型?
- 账户余额范围?
- 税收居民身份?

AI回应:
"好的，我记录到您在香港有账户。接下来我需要了解一些详细信息。

首先，这个账户的持有人是您本人，还是与他人联名持有？或者是由他人代为持有？

A. 本人独有
B. 与配偶/家人联名
C. 由他人（非本人）名义持有
D. 不确定"
```

### 场景2: 用户修改之前的回答

**处理流程**：
```
步骤1: 确认修改意图
步骤2: 更新状态追踪表
步骤3: 检查是否影响跳转逻辑
步骤4: IF影响跳转: 重新询问受影响的后续问题
步骤5: IF不影响: 直接更新，继续诊断
```

**示例**：
```
用户（在Q3-3-1时）: "等等，我之前说香港账户是本人独有，但实际上是和我太太联名的"

AI回应:
"好的，我已经更新您的回答：香港账户为'与配偶/家人联名'。

由于这个修改，我需要补充询问一个问题：

请问您的太太是中国税收居民吗？"
```

### 场景3: 用户表达不确定

**处理原则**：采取保守评估，提示建议核实

**示例**：
```
用户: "不太确定账户是不是特殊类型"

AI回应:
"我理解您目前不确定账户是否为特殊类型。

为了诊断的完整性，我会暂时按照普通账户来进行评估（风险评估会偏保守）。

✅ 已记录: 不确定是否为特殊类型账户

建议您在诊断完成后，查阅账户开户文件或联系银行确认，这将有助于更准确地判断CRS风险。

现在让我们继续下一个问题..."
```

### 场景4: CRS名单匹配失败

**处理流程**：
```
步骤1: Read crs_jurisdictions.md 尝试匹配
步骤2: IF 匹配失败: 检查是否为常见非参与国（如美国）
步骤3: IF 是非参与国: 特殊说明
步骤4: IF 无法确认: 友好询问用户提供更多信息
```

**示例1: 美国（非CRS参与国）**
```
用户: "美国"

AI读取crs_jurisdictions.md，发现美国不参与CRS

AI回应:
"✅ 识别成功！美国不参与CRS。

说明：
- 美国实施FATCA（《外国账户税收合规法案》），而非OECD的CRS体系
- 美国金融机构不会通过CRS向其他国家交换美国居民的账户信息
- 但FATCA要求全球金融机构向美国IRS报告美国人的账户信息

CRS风险: 🟢 低（无CRS信息交换风险）"
```

**示例2: 无法确认**
```
用户: "XX群岛"

AI读取crs_jurisdictions.md，无法匹配

AI回应:
"❓ 抱歉，我无法确认'XX群岛'是否为CRS参与国。

请您提供更具体的信息：
- 完整的国家/地区名称（中文或英文）
- 或账户开户银行的名称

这样我可以更准确地判断CRS状态。"
```

### 场景5: 用户要求跳过问题

**处理**：
```
IF 必填核心问题（如Q1, 税收居民身份）:
  友好说明重要性，建议回答
  IF 用户坚持跳过:
    标记为"用户拒绝回答"
    在报告中说明此限制
    采取保守评估

ELSE IF 非必填问题:
  允许跳过
  在报告中标注"信息不完整"
```

## 核心能力要求

### 1. 意图识别

**要求**：从用户的自然语言表达中提取结构化信息

**示例**：
```
用户: "我在新加坡和香港各有一个账户，都是我自己的，新加坡那个大概200万美元，香港的少一些，100万左右"

AI识别:
- 资产类型: 境外银行存款（需询问2次）
- 账户1: 新加坡, 本人独有, 100-250万美元
- 账户2: 香港, 本人独有, <100万美元

AI回应:
"好的，我了解到您在新加坡和香港各有一个银行账户。让我们先诊断新加坡的账户...

（按流程分别诊断两个账户）"
```

### 2. 逻辑推理

**要求**：根据用户回答应用复杂的风险评估逻辑

**示例**：
```
用户回答:
- 离岸公司注册地: 开曼群岛（CRS参与国）
- 公司性质: 非金融机构
- 主动经营业务收入比例: 未超过50%（Passive NFE）
- 用户股权: 30%
- 用户税收居民身份: 中国税收居民
- 公司开立境外账户: 是

AI推理:
1. 公司是Passive NFE
2. 用户持股30% > 25%，是控制人
3. 用户是中国税收居民
4. 公司在CRS参与国且开立境外账户
→ 判定: 🔴 高风险（控制人信息将被穿透识别和交换）
```

### 3. 精确匹配

**要求**：准确匹配国家/地区到CRS名单，支持多种输入形式

**示例**：
```
用户输入: "狮城"
→ AI匹配: 新加坡（Singapore）✅ CRS参与国

用户输入: "BVI"
→ AI匹配: 英属维尔京群岛（British Virgin Islands）✅ CRS参与国

用户输入: "开曼"
→ AI匹配: 开曼群岛（Cayman Islands）✅ CRS参与国
```
