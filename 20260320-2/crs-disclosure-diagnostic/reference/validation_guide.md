# CRS诊断答案校验指南

## 文件说明

本文件定义CRS信息披露诊断过程中用户回答的校验规则和标准。

**版本**: v1.0
**用途**: 供大模型验证用户输入的完整性、合法性和逻辑一致性

## 使用指南（供大模型使用）

### 校验原则

1. **友好提示**: 当发现问题时，使用友好、简洁的语言提示用户
2. **不过度严格**: 允许用户用自然语言表达，不要求完全标准化
3. **智能识别**: 使用意图识别提取关键信息，而非强制固定格式
4. **逐步收集**: 如果信息不完整，分步骤询问缺失的部分
5. **上下文理解**: 结合之前的回答理解当前回答

### 校验流程

```
步骤1: 接收用户回答
  ↓
步骤2: 提取关键信息（意图识别）
  ↓
步骤3: 格式校验（数据类型、范围检查）
  ↓
步骤4: 完整性校验（必填信息是否齐全）
  ↓
步骤5: 逻辑一致性校验（与之前回答是否矛盾）
  ↓
步骤6: 合理性校验（常识检查）
  ↓
步骤7: 如有问题，友好提示用户补充或修正
  ↓
步骤8: 验证通过，继续下一问题
```

## 一、按问题类型的校验规则

### 1.1 单选题校验

**基本规则**:
- 用户必须选择一个选项
- 可以接受选项的文本、字母标识（A/B/C/D）或意图表达

**校验步骤**:

```
1. 检查用户是否提供了回答
2. 尝试匹配用户输入到选项列表
   - 精确匹配选项文本
   - 匹配字母标识（A/B/C/D）
   - 意图识别（如"我是"→"是"）
3. 如果匹配成功，记录选择
4. 如果匹配失败或模糊，友好提示用户澄清
```

**示例**:

| 用户输入 | 识别结果 | 处理 |
|---------|---------|------|
| "A" | 选项A | ✅ 通过 |
| "本人独有" | 选项A（假设） | ✅ 通过 |
| "我自己的" | 意图识别 → "本人独有" | ✅ 通过 |
| "不太清楚" | 模糊 | ⚠️ 提示: "请从以下选项中选择：A. 本人独有 B. 联名..." |
| 空白/无效输入 | 无回答 | ❌ 提示: "请回答这个问题以继续诊断" |

### 1.2 多选题校验

**基本规则**:
- 用户可以选择一个或多个选项
- 可以接受列表形式或自然语言表达

**校验步骤**:

```
1. 检查用户是否提供了回答
2. 提取所有被选择的选项
   - 解析逗号分隔的列表
   - 解析"和"、"、"等连接词
   - 识别字母标识（A, B, C...）
3. 验证每个选项是否有效
4. 如果有无效选项，提示用户
5. 如果没有选择任何选项，提示用户至少选择一个
```

**示例**:

| 用户输入 | 识别结果 | 处理 |
|---------|---------|------|
| "A, B, C" | 选项A、B、C | ✅ 通过 |
| "境外银行存款和离岸公司" | 识别为2个选项 | ✅ 通过 |
| "银行存款、证券" | 智能匹配 | ✅ 通过（如果能匹配） |
| "都有" | 模糊 | ⚠️ 提示: "请具体说明您拥有哪些资产类型" |
| "没有" | 无选择 | ⚠️ 特殊处理（可能需要结束诊断） |

### 1.3 文本输入校验

**基本规则**:
- 用户必须提供有意义的文本输入
- 根据问题类型有不同的校验标准

**分类校验**:

#### A. 国家/地区名称（Q1-1, Q2-1-1, Q3-1-1等）

**校验规则**:
```
1. 非空检查: 用户必须输入内容
2. 合理性检查: 输入应该像是国家/地区名称
   - 长度: 2-50字符
   - 不应是纯数字或特殊符号
3. CRS匹配尝试:
   - 尝试在crs_jurisdictions.md中匹配
   - 如果匹配失败，询问用户澄清
4. 记录原始输入和匹配结果
```

**示例**:

| 用户输入 | 识别结果 | 处理 |
|---------|---------|------|
| "香港" | 匹配成功 → "中国香港" | ✅ 通过，记录为CRS参与国 |
| "Singapore" | 匹配成功 → "新加坡" | ✅ 通过，记录为CRS参与国 |
| "狮城" | 匹配成功 → "新加坡" | ✅ 通过（别名匹配） |
| "XX国" | 匹配失败 | ⚠️ 提示: "抱歉，我无法确认'XX国'是否为CRS参与国，请提供完整名称" |
| "123" | 不合理 | ❌ 提示: "请输入有效的国家/地区名称" |
| 空白 | 无输入 | ❌ 提示: "请说明账户开立的国家或地区" |

#### B. 比例/数值（Q3-3-1股权比例等）

**校验规则**:
```
1. 非空检查
2. 提取数值:
   - 识别百分比（如"25%"）
   - 识别小数（如"0.25"）
   - 识别文字描述（如"四分之一"）
3. 范围检查:
   - 股权比例应在0-100%之间
   - 提示不合理的数值
4. 提取控制权描述:
   - 识别"有控制权"、"无控制权"等表达
```

**示例**:

| 用户输入 | 识别结果 | 处理 |
|---------|---------|------|
| "30%，有控制权" | 比例=30%, 控制权=是 | ✅ 通过 |
| "我持有25个点" | 比例=25% | ✅ 通过（意图识别） |
| "大概三成左右" | 比例≈30% | ✅ 通过（智能识别） |
| "很少，没有发言权" | 比例不明，控制权=否 | ⚠️ 提示: "请提供大概的股权比例（如10%、20%）" |
| "150%" | 超出范围 | ❌ 提示: "股权比例应在0-100%之间" |

#### C. 身份/角色描述（Q1-2-2联名人信息、Q3-3-3其他控制人等）

**校验规则**:
```
1. 非空检查
2. 提取身份信息:
   - 关系（配偶/子女/父母/合作伙伴等）
   - 税收居民身份（是/否）
3. 完整性检查:
   - 如果提到多个人，每个人的信息是否完整
4. 允许用户多轮补充信息
```

**示例**:

| 用户输入 | 识别结果 | 处理 |
|---------|---------|------|
| "配偶，是中国税收居民" | 关系=配偶, 税收居民=是 | ✅ 通过 |
| "我太太，她也是中国人" | 关系=配偶, 税收居民=是（推断） | ✅ 通过（智能识别） |
| "儿子" | 关系=子女, 税收居民=? | ⚠️ 追问: "请问您的儿子是中国税收居民吗？" |
| "有两个联名人" | 数量=2, 详情=? | ⚠️ 追问: "请分别说明每个联名人的身份和税收居民状态" |
| "没有" | 无其他控制人 | ✅ 通过 |

#### D. 开放式描述（Q5-2-A-1信托资产等）

**校验规则**:
```
1. 非空检查（除非"无"或"没有"是合理回答）
2. 意图识别:
   - 提取所有提到的资产类型
   - 匹配到预设选项列表
3. 允许自由表达，不强制格式
4. 如果无法理解，友好询问澄清
```

**示例**:

| 用户输入 | 识别结果 | 处理 |
|---------|---------|------|
| "现金、公司股权" | 资产类型=现金+股权 | ✅ 通过 |
| "主要是钱和房子" | 资产类型=现金+房产 | ✅ 通过（意图识别） |
| "一些投资" | 模糊 | ⚠️ 追问: "请具体说明投资的类型（如股票、债券、房产等）" |
| "很多东西" | 过于模糊 | ⚠️ 追问: "请从以下类型中选择或说明：现金、公司股权、房产、证券..." |

## 二、逻辑一致性校验

### 2.1 税收居民身份一致性

**规则**: 同一个人的税收居民身份应在整个诊断过程中保持一致

**检查点**:
- Q1-2-1（银行存款）
- Q2-1-2-1（证券投资）
- Q3-3-2（离岸公司）
- Q4-2-1-1（境外保单）
- Q5-1-2（离岸信托）

**校验逻辑**:

```
IF 用户在不同问题中对"是否是中国税收居民"给出矛盾回答:
  THEN 友好提示:
    "我注意到您在回答银行存款问题时说是中国税收居民，
     但在回答离岸公司问题时说不是。
     请确认：您目前是中国税收居民吗？"
  
  等待用户澄清，更新所有相关回答
```

### 2.2 资产类型一致性

**规则**: 用户在Q1选择的资产类型，应该后续会被询问；未选择的类型不应该被询问

**检查点**:
- Q1（初始资产选择）
- 后续各资产类型的详细问题

**校验逻辑**:

```
IF 用户在Q1中选择了"境外银行存款":
  THEN 必须询问Q1-1至Q1-4

IF 用户在Q1中未选择"境外银行存款":
  THEN 跳过Q1-1至Q1-4，报告中标记为"未涉及"
```

### 2.3 跳转逻辑一致性

**规则**: 严格按照questions.md中定义的跳转逻辑执行

**示例**:

```
Q1-2 → 根据用户选择跳转:
  "本人独有" → Q1-2-1
  "与配偶/家人联名" → Q1-2-2
  "由他人持有" → Q1-2-3
  "不确定" → Q1-2-3

IF 用户选择"本人独有":
  THEN 下一问必须是Q1-2-1
  NOT Q1-2-2 或 Q1-2-3（除非用户后来修改了回答）
```

### 2.4 金额/比例合理性

**规则**: 数值应该在合理范围内

**检查点**:

| 字段 | 合理范围 | 不合理提示 |
|-----|---------|-----------|
| 股权比例 | 0-100% | "股权比例应在0-100%之间" |
| 账户余额 | 提供范围选项 | （单选题，预设范围） |
| 年龄（如涉及） | 18-120岁 | "请提供有效的年龄" |

## 三、完整性校验

### 3.1 必填问题检查

**规则**: 某些核心问题必须有明确回答才能继续

**必填问题列表**:

| 问题ID | 问题内容 | 缺失时的提示 |
|--------|---------|-------------|
| Q1 | 资产类型选择 | "请选择您拥有的资产类型以开始诊断" |
| Q1-1 | 银行账户开立地 | "请说明银行账户的开立国家/地区" |
| Q1-2-1 | 税收居民身份 | "请确认您是否是中国税收居民" |
| Q3-3-1 | 股权比例及控制权 | "请说明您的股权比例和控制权情况" |

**校验逻辑**:

```
FOR 每个必填问题:
  IF 用户回答为空 OR 无效:
    THEN 友好提示用户回答
    等待用户补充
    不允许跳过此问题
```

### 3.2 信息完整性检查

**规则**: 对于需要多个信息点的问题，确保所有信息都被收集

**示例1: Q1-2-2 联名持有人信息**

```
必需信息:
- 联名持有人数量
- 每个联名持有人的身份（关系）
- 每个联名持有人的税收居民身份

校验:
IF 用户只提供了身份，未提供税收居民身份:
  THEN 追问: "请问{身份}是中国税收居民吗？"

IF 用户提到"有两个联名人"但只描述了一个:
  THEN 追问: "请继续说明第二个联名持有人的情况"
```

**示例2: Q3-3-1 股权比例及控制权**

```
必需信息:
- 股权比例（数值或范围）
- 控制影响力（是/否/描述）

校验:
IF 用户只提供股权比例，未提及控制权:
  THEN 追问: "您是否对该公司的重大决策有控制影响力？"

IF 用户只说"有控制权"，未提供股权比例:
  THEN 追问: "请补充说明您持有的股权比例（大约百分比）"
```

## 四、友好提示模板

### 4.1 格式错误提示

**模板**:
```
❌ 抱歉，我无法理解您的回答。

请{具体要求}。

例如：{示例}
```

**示例**:
```
❌ 抱歉，我无法理解"xx"是否对应某个选项。

请从以下选项中选择一个：
A. 本人独有
B. 与配偶/家人联名
C. 由他人（非本人）名义持有
D. 不确定

您可以直接输入字母（如"A"）或选项文字（如"本人独有"）。
```

### 4.2 信息不完整提示

**模板**:
```
✏️ 感谢您的回答！为了更准确地评估，我还需要了解{缺失信息}。

请补充：{具体问题}
```

**示例**:
```
✏️ 感谢您的回答！为了更准确地评估，我还需要了解联名持有人的税收居民身份。

请补充：您的配偶是中国税收居民吗？
```

### 4.3 逻辑不一致提示

**模板**:
```
🤔 我注意到您之前提到{之前的回答}，但现在说{当前的回答}。

请确认：{澄清问题}
```

**示例**:
```
🤔 我注意到您之前提到您是中国税收居民，但现在说持有人（您配偶）不是中国税收居民。

请确认：
1. 您本人是中国税收居民吗？
2. 您配偶是中国税收居民吗？
```

### 4.4 无效输入提示

**模板**:
```
⚠️ 您输入的{内容}似乎不符合预期。

{原因说明}

请{正确做法}。
```

**示例**:
```
⚠️ 您输入的股权比例"150%"似乎不符合预期。

股权比例应该在0-100%之间。

请重新输入您持有的股权比例（例如：30%、50%等）。
```

### 4.5 模糊回答引导

**模板**:
```
💡 我理解您的意思，但为了准确诊断，需要更具体的信息。

您是想表达{推测意图}吗？

或者，请从以下选项中选择：{选项列表}
```

**示例**:
```
💡 我理解您的意思，但为了准确诊断，需要更具体的信息。

您是想表达"账户余额低于100万美元"吗？

或者，请从以下选项中选择：
A. 低于100万美元
B. 100万至250万美元
C. 超过250万美元
```

## 五、特殊情况处理

### 5.1 用户要求跳过问题

**处理原则**: 区分必填和非必填问题

**必填问题（如Q1、税收居民身份）**:
```
提示: "抱歉，这个问题是诊断的关键信息，需要您回答才能继续。如果您不确定，我可以帮助您理解这个问题。"

如果用户坚持跳过: "好的，我理解。但请注意，缺少这个信息可能导致诊断结果不准确。"

标记为"用户拒绝回答"，在报告中说明此限制
```

**非必填问题（如某些细节问题）**:
```
提示: "好的，我们可以暂时跳过这个问题。如果需要更准确的评估，您可以稍后补充这个信息。"

继续下一问题
```

### 5.2 用户修改之前的回答

**处理流程**:
```
1. 确认用户的修改意图
   "您想修改之前关于{问题}的回答吗？"

2. 记录新的回答

3. 检查是否影响后续问题的跳转逻辑
   IF 修改影响跳转:
     THEN 友好说明: "由于您修改了回答，我需要重新询问几个相关问题。"
     重新询问受影响的问题

4. 更新状态追踪表
```

### 5.3 用户输入多个答案

**场景**: 用户在一个回答中提供多个问题的信息

**处理**:
```
1. 使用意图识别提取所有信息
   "我理解您提到了以下信息：
    - {信息1}
    - {信息2}
    - {信息3}"

2. 逐一确认
   "这样理解对吗？我会根据这些信息继续诊断。"

3. 记录所有信息，跳过已回答的问题

4. 如果某些信息不完整，继续询问缺失部分
```

**示例**:
```
用户: "我在香港有个银行账户，是我自己的，余额大概200多万美元，我是中国税收居民"

AI识别:
- Q1-1: 香港
- Q1-2: 本人独有
- Q1-3-2: 100万至250万美元
- Q1-2-1: 是中国税收居民

AI回应:
"✅ 谢谢您的详细信息！我已记录：
- 账户开立地：香港
- 持有人类型：本人独有
- 账户余额：100万至250万美元
- 税收居民身份：是中国税收居民

现在请继续回答下一个问题：这个账户属于以下哪种类型？..."
```

### 5.4 用户表达不确定

**处理原则**: 采取保守评估，并友好建议核实

**回应模板**:
```
"我理解您目前不确定{具体内容}。

为了诊断的完整性，我会按照{保守假设}来进行评估。

建议您在诊断完成后，{建议措施}。"
```

**示例**:
```
用户: "不太确定是不是特殊类型账户"

AI回应:
"我理解您目前不确定账户是否为特殊类型。

为了诊断的完整性，我会暂时按照普通账户来进行评估（风险评估会偏保守）。

建议您在诊断完成后，查阅账户开户文件或联系银行确认账户类型，这将有助于更准确地判断CRS风险。"
```

## 六、校验通过标准

### 6.1 单个回答通过标准

**标准**:
```
✅ 格式正确: 回答格式符合问题类型要求
✅ 内容有效: 回答内容在有效范围内，非空且有意义
✅ 可识别: AI能够明确理解用户意图并提取关键信息
✅ 逻辑自洽: 回答与之前的回答不矛盾
```

### 6.2 整体诊断完成标准

**标准**:
```
✅ 所有必填问题已回答
```

## 七、校验失败处理流程

### 7.1 轻微问题（不阻断诊断）

**示例**: 用户描述稍微模糊，但AI能推断意图

**处理**:
```
1. 记录AI的推断结果
2. 向用户确认: "我理解您的意思是{推断结果}，对吗？"
3. 如果用户确认，继续；如果用户否认，重新询问
4. 在报告中标注推断的部分
```

### 7.2 中等问题（需要补充信息）

**示例**: 用户回答缺少关键信息

**处理**:
```
1. 明确指出缺失的信息
2. 友好询问补充
3. 等待用户补充
4. 重新校验
5. 如果用户多次无法提供，询问是否跳过（参见5.1）
```

### 7.3 严重问题（阻断诊断）

**示例**: 用户拒绝回答必填核心问题（如Q1资产类型选择）

**处理**:
```
1. 明确说明该问题的重要性
2. 尝试解释为何需要这个信息
3. 如果用户仍拒绝，友好说明无法继续诊断
4. 提供替代方案（如"您可以稍后准备好信息后再来诊断"）
5. 记录终止原因
```
