# 04 CRS Skill 方案三（下策）：最小化快速落地方案

**文档编号**：SOL-CRS-SKILL-003  
**方案级别**：下策  
**撰写日期**：2026-03-21  
**适用场景**：极度追求快速上线，可接受较低的用户体验和有限的业务覆盖范围

---

## 一、方案概述

本方案以最快速度交付一个可运行的 CRS 风险评估 Skill，放弃严格的逐题引导和主动意图分析，改为按资产类型分块收集信息（每类资产一次性列出所有问题），用户填写完成后统一处理。判断逻辑简化为对核心规则的直接套用，不处理多层架构和复杂边界情况。报告生成以填充模板为主，内容较为简短。

本方案的核心价值是：在最短时间内（约 0.5-1 个工作日）交付一个能够运行的版本，适合用作快速验证业务概念（POC）或内部测试，不建议作为正式面向用户的产品方案。

## 三、交互设计

### 3.1 按资产类型分块提问

放弃逐题引导原则，改为按资产类型分块呈现问题。每次以一个资产类型为单位，列出该类型下所有需要收集的信息，引导用户在一次回复中提供完整信息。

示例：
> "好的，我们来了解一下您的海外银行账户情况。请您告诉我以下几点：① 账户开在哪个国家或地区？② 是个人账户还是公司/信托账户？③ 账户余额大约在什么范围？（如果有多个账户，可以逐一描述）"

这种方式效率更高，但用户需要一次处理多个问题，体验较逐题引导更为生硬。

### 3.2 简单关键词匹配

不做完整的语义意图分析。当用户的回答明显包含了所需信息时，智能体直接提取；当用户的回答明显不完整时，整体再次询问该块信息（而不是精准追问缺失字段）。

示例（不完整时）：
> "您刚才的信息里我没有找到账户所在地，能否重新告诉我：① 账户在哪个国家？② 是个人还是公司账户？"

### 3.3 无完整性校验

不做字段级的完整性校验。若用户的回答信息不完整且未主动追问，则在判断时将缺失字段处理为"不清楚"，对应资产标注为"存在风险"。

### 3.4 无中途知识问答支持

若用户提出 CRS 背景性问题，智能体简短回应后立即引导用户回到资产信息填写，不提供延伸解答。

## 四、功能模块详细设计

### 4.1 资产覆盖范围（三类核心资产）

本方案仅覆盖最常见的三类资产，去掉了在实际用户中持有比例较低、判断逻辑最复杂的两类：

**保留：银行存款 / 金融账户、上市公司股票、信托**（这三类在高净值人群中持有比例最高，也是 CRS 交换的主要标的）

**暂不覆盖：非上市公司股权、保险保单**（判断逻辑涉及大量架构穿透分析，实现复杂度最高；可在后续版本补充）

若用户提到持有非上市股权或保险，智能体告知该版本暂未覆盖，建议进行专项咨询，并继续处理其他资产类型。

### 4.2 三类资产问卷（块状收集）

**资产一：银行存款 / 金融账户**

一次性收集：账户所在地 / 账户类型（个人/公司/信托） / 若为公司或信托账户，其主营业务是投资还是经营

**资产二：上市公司股票**

一次性收集：股票所在地 / 持有方式 / 若为架构持有，架构主营业务

**资产三：信托**

一次性收集：信托设立地 / 信托类型 / 用户角色 / 信托是否主要投资金融资产

### 4.3 判断逻辑（简化规则）

不分五条链条，统一使用三个核心规则：

规则一——所在地检查：资产所在地是否为 CRS 成员国。若是，进入规则二；若否，结论为"不会交换"。

规则二——架构性质检查：若通过公司或信托持有，判断该实体是否为投资机构（被动收入超过 50% 或主营业务为投资理财）。若是投资机构，结论为"会交换"；若是实际经营机构，结论为"存在风险，建议进一步分析"。

规则三——个人直接持有：若为个人直接持有，且资产位于 CRS 成员国，结论为"会交换"。

对于任何信息不完整或逻辑无法覆盖的情况，统一标注为"存在风险，建议进一步分析"。

注意：本方案的判断规则经过高度简化，部分场景（尤其是架构持有情况）的判断结论可能不够精确，需在报告中通过免责声明进行风险提示。

### 4.4 报告生成（简版格式）

报告结构简化为两个部分：

第一部分——资产清单与结论：逐项列出每类资产的名称、所在地、结论（会交换/不会交换/存在风险）。

第二部分——重要声明：说明本报告基于简化规则生成，可能存在遗漏和误判，建议结合专业税务顾问意见进行决策。

不生成"综合建议"章节，以降低生成复杂度。

### 4.5 报告输出

仅支持聊天窗口展示，无邮件发送。

## 六、方案的局限性说明

本方案存在以下明显局限，在决策时必须明确知晓：

用户体验偏差：一次性列出多个问题会让用户感受到压力，对不熟悉结构化填写的用户尤其不友好，整体对话体验显著低于上策和中策。

业务覆盖不完整：保险和非上市股权是高净值人群常见资产类型，跳过这两类会导致风险评估不全面，用户可能误以为自己没有风险。

判断准确性有限：简化的判断规则在架构持有等复杂场景下可能给出不准确的结论，存在误导风险。

不适合正式上线：本方案适合作为概念验证（POC）版本，用于内部测试流程跑通，不建议直接面向真实用户提供服务。
