# CRS信息披露诊断报告 - 输出模板说明

## 模板概览

CRS信息披露诊断智能体输出的诊断报告采用结构化Markdown格式，包含6个主要部分。本文档详细说明各部分的输出规范和填充逻辑。

---

## 一、诊断概要

### 模板结构

```markdown
## 一、诊断概要

- 账套名称：{account}
- 诊断时间：{timestamp}
- 涉及资产类型：{asset_types}
```

### 字段说明

| 占位符 | 说明 | 数据来源 | 示例 |
|--------|------|----------|------|
| `{account}` | 账套名称 | 用户创建账套时指定 | "张先生_2025" |
| `{timestamp}` | 诊断完成时间 | 系统生成报告时的时间戳 | "2025-03-10 14:30:25" |
| `{asset_types}` | 涉及的资产类型列表 | Q1 用户多选的资产类型 | "境外银行存款、离岸公司、境外保单" |

### 输出示例

```markdown
## 一、诊断概要

- 账套名称：张先生_2025
- 诊断时间：2025-03-10 14:30:25
- 涉及资产类型：境外银行存款、离岸公司、境外保单
```

## 二、资产总览

### 模板结构

```markdown
## 二、资产总览

| 资产类型 | 是否涉及 | 风险等级 |
|----------|----------|----------|
| 境外银行存款 | {bank} | {bank_risk} |
| 境外证券投资 | {securities} | {securities_risk} |
| 离岸公司 | {company} | {company_risk} |
| 境外保单 | {insurance} | {insurance_risk} |
| 离岸信托 | {trust} | {trust_risk} |
```

### 字段说明

#### 是否涉及字段

- **填充值**: "是" / "否"
- **判定逻辑**: 基于 Q1 用户是否选择该资产类型

#### 风险等级字段

- **填充值**: "🔴 高风险" / "🟡 中风险" / "🟢 低风险" / "-"
- **判定逻辑**:

| 风险等级 | 判定条件 |
|---------|---------|
| 🔴 高风险 | ① 用户是中国税收居民 <br> ② 账户/资产在CRS参与国/地区 <br> ③ 余额超过100万美元或涉及控制权 |
| 🟡 中风险 | ① 用户是中国税收居民 <br> ② 账户/资产在CRS参与国/地区 <br> ③ 余额低于100万美元或间接持有 |
| 🟢 低风险 | ① 账户在非CRS国家（如美国）<br> ② 或用户不是中国税收居民 <br> ③ 或属于豁免账户 |
| - | 用户未选择该资产类型 |

### 输出示例

```markdown
## 二、资产总览

| 资产类型 | 是否涉及 | 风险等级 |
|----------|----------|----------|
| 境外银行存款 | 是 | 🔴 高风险 |
| 境外证券投资 | 否 | - |
| 离岸公司 | 是 | 🟡 中风险 |
| 境外保单 | 是 | 🟢 低风险 |
| 离岸信托 | 否 | - |
```

## 三、各资产信息披露义务判定

### 3.1 境外银行存款

#### 模板结构

```markdown
### 3.1 境外银行存款

{bank_section}
```

#### 输出内容（`{bank_section}` 生成逻辑）

如果用户**未选择**该资产类型：

```markdown
*未涉及此类资产*
```

如果用户**选择**该资产类型，输出以下结构：

```markdown
#### 基本信息

- **开户地区**: {bank_country}
- **持有人类型**: {bank_holder_type}
- **税收居民身份**: {bank_is_china_resident}
- **账户类型**: {bank_account_type}
- **账户余额范围**: {bank_balance_range}
- **是否为特殊账户**: {bank_special_type}

#### CRS信息披露义务判定

**判定结果**: {disclosure_obligation}

**理由**:

{reasoning}

#### 信息交换预期

{exchange_expectation}
```

#### 字段填充规则

| 字段 | 数据来源 | 说明 |
|-----|---------|------|
| `{bank_country}` | Q1-1 | 用户填写的开户国家/地区 |
| `{bank_holder_type}` | Q1-2 | 本人独有/联名/代持/不确定 |
| `{bank_is_china_resident}` | Q1-2-1 | 是/否 |
| `{bank_account_type}` | Q1-3-1 | 活期/定期/信用卡/其他 |
| `{bank_balance_range}` | Q1-3-2 | 低于100万/100-250万/超过250万美元 |
| `{bank_special_type}` | Q1-4 | 是/否/不确定 |
| `{disclosure_obligation}` | 综合判定 | ✅ 有披露义务 / ⚠️ 可能有披露义务 / ❌ 无披露义务 |

#### `{reasoning}` 生成逻辑

根据以下条件综合判定：

1. **CRS参与国判定**
   - 如果 `bank_country` 在CRS参与国/地区名单中 → "开户地区是CRS参与国"
   - 否则 → "开户地区不是CRS参与国（如美国），无CRS披露义务"

2. **税收居民身份**
   - 如果 `bank_is_china_resident` = "是" → "账户持有人是中国税收居民"
   - 否则 → "账户持有人不是中国税收居民，中国税务机关无法收到CRS信息"

3. **账户余额判定**
   - 如果 `bank_balance_range` = "超过250万美元" → "账户余额超过250万美元，属于高净值账户，银行会加强尽职调查"
   - 如果 `bank_balance_range` = "100万至250万美元" → "账户余额在100-250万美元之间，需完成尽职调查"
   - 否则 → "账户余额低于100万美元，但仍需完成尽职调查"

4. **豁免账户判定**
   - 如果 `bank_special_type` = "是" → "该账户可能属于CRS豁免账户（如退休金账户），需核实具体情况"

#### `{exchange_expectation}` 生成逻辑

根据综合判定结果：

- **有披露义务**: 
  ```
  开户银行将在每年4-5月前，向{bank_country}当地税务机关报送账户信息，包括：
  - 账户持有人姓名、地址、税收居民国
  - 账户号码
  - 账户余额或价值
  - 利息、股息等收入金额
  
  {bank_country}税务机关将在当年9月前，通过CRS自动交换机制，将上述信息传递给中国税务机关。
  ```

- **无披露义务**:
  ```
  该账户不属于CRS信息交换范围，无需担心信息披露。
  ```

#### 完整输出示例

```markdown
### 3.1 境外银行存款

#### 基本信息

- **开户地区**: 香港
- **持有人类型**: 本人独有
- **税收居民身份**: 是中国税收居民
- **账户类型**: 活期/储蓄账户
- **账户余额范围**: 超过250万美元
- **是否为特殊账户**: 不是特殊类型账户

#### CRS信息披露义务判定

**判定结果**: ✅ 有披露义务

**理由**:

1. 开户地区香港是CRS参与地区，已与中国签署税收情报自动交换协议
2. 账户持有人是中国税收居民
3. 账户余额超过250万美元，属于高净值账户，银行会加强尽职调查
4. 该账户不属于豁免账户

综合判定：该账户符合CRS信息披露要求，开户银行有义务向香港税务局报送账户信息。

#### 信息交换预期

开户银行将在每年4-5月前，向香港税务局报送账户信息，包括：
- 账户持有人姓名、地址、税收居民国
- 账户号码
- 账户余额或价值
- 利息、股息等收入金额

香港税务局将在当年9月前，通过CRS自动交换机制，将上述信息传递给中国税务机关。
```

#### `{crs_classification}` 生成逻辑

根据问答结果判定离岸公司的CRS分类：

| 判定条件 | CRS分类 |
|---------|---------|
| Q3-1-2 = "是金融机构" | **金融机构（FI）**<br>需作为报送金融机构，履行CRS尽职调查义务 |
| Q3-2-2 = "主动经营业务比例超50%" | **积极非金融机构（Active NFE）**<br>相对风险较低 |
| Q3-2-2 = "主动经营业务比例未超50%" | **消极非金融机构（Passive NFE）**<br>需穿透至实际控制人 |
| Q3-2-2 = "不清楚" | **待确认**<br>建议核实财务数据后再判定 |

#### `{compliance_advice}` 生成逻辑

根据CRS分类给出建议：

- **消极NFE + 开立境外账户 + 控制人是中国税收居民**:
  ```
  🔴 高风险情况：
  
  1. 该离岸公司属于消极NFE，其开立的境外金融账户将被穿透至实际控制人层面
  2. 开户金融机构将要求您填写《实际控制人声明》或类似文件
  3. 如您是中国税收居民，账户信息将通过CRS交换至中国税务机关
  
  建议措施：
  - 确保已在开户时如实申报税收居民身份
  - 准备好证明公司经营性质的材料（财务报表、业务合同等）
  - 考虑将公司调整为积极NFE（增加主动经营业务比例至50%以上）
  - 咨询专业税务顾问，评估是否需要主动进行税务申报
  ```

- **积极NFE**:
  ```
  🟢 相对低风险：
  
  该公司属于积极NFE，即使开立境外金融账户，金融机构通常无需穿透至实际控制人层面。但仍需注意：
  - 保持主动经营业务比例持续超过50%
  - 每年更新财务数据，以便向开户机构证明积极NFE身份
  - 如公司性质发生变化（如转为投资控股），需及时通知开户机构
  ```

```markdown
- **注册地区**: 开曼群岛
- **是否为金融机构**: 不是金融机构
- **经营性质**: 主要作为投资控股工具（主要资产是持有的其他公司股权、股票、债券等）

- **是否开立境外金融账户**: 是，在开曼群岛本地银行开立有账户

#### CRS分类判定

**分类结果**: 🔴 消极非金融机构（Passive NFE）

**判定依据**:

1. 该公司不是金融机构
2. 主要资产为投资控股（股权、股票、债券等）
3. 主动经营业务收入比例和资产比例均未超过50%
4. 根据CRS规则，该公司属于消极NFE

**判定结果**: ✅ 有披露义务（需穿透至实际控制人）

**理由**:

1. 开曼群岛是CRS参与地区
2. 该公司属于消极NFE，开立的境外金融账户需被穿透
3. 实际控制人（您及配偶）均为中国税收居民
4. 根据CRS穿透规则，开户银行需识别并报送实际控制人信息

#### 信息交换预期

开曼群岛的开户银行将：
1. 要求填写《实际控制人声明》，识别您和配偶为实际控制人
2. 确认您和配偶均为中国税收居民
3. 向开曼群岛税务机关报送以下信息：
   - 实际控制人姓名、地址、税收居民国（中国）
   - 账户号码及公司名称
   - 账户余额或价值
   - 利息、股息等收入金额

开曼群岛税务机关将通过CRS自动交换机制，将上述信息传递给中国税务机关。

#### 合规建议

🔴 高风险情况：

1. 该离岸公司属于消极NFE，其开立的境外金融账户将被穿透至实际控制人层面
2. 开户金融机构将要求您填写《实际控制人声明》或类似文件
3. 如您是中国税收居民，账户信息将通过CRS交换至中国税务机关

建议措施：
- 确保已在开户时如实申报税收居民身份
- 准备好证明公司经营性质的材料（财务报表、业务合同等）

- 主动评估是否需要在中国进行受控外国企业（CFC）申报
```

   ```bash
   python scripts/engine.py report-data {账套名称}
   ```

   ```json
       "1": ["境外银行存款", "离岸公司", "境外保单"],
   ```

   - 列举关键风险点
   - 给出分级合规建议

   保存到 `sessions/{账套名称}/reports/{timestamp}.md`

### 大模型输出要求

1. **专业准确**：基于CRS规则和中国税法，准确判定信息披露义务
2. **通俗易懂**：避免过多专业术语，用通俗语言解释复杂概念

### 关键术语解释

报告中使用专业术语时，建议附加通俗解释：

- **CRS（共同申报准则）**："全球税务透明化"机制，各国税务机关自动交换非居民金融账户信息
- **消极NFE**：主要从事投资活动（如持有股权、债券）的公司，而非实体经营（如生产、贸易）
- **积极NFE**：有实质性经营活动的公司（如制造业、服务业）
- **穿透**：金融机构不仅报送账户名义持有人信息，还要识别并报送实际控制人信息
- **实际控制人**：对账户、公司、信托有实质性控制权的自然人（通常持股超过25%或有否决权）
- **受控外国企业（CFC）**：中国税收居民控制的境外企业，其未分配利润可能需要在中国纳税
